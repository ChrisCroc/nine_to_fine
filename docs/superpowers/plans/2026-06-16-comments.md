# Comments on Outfit — Implementation Plan

> **Coach-mode note:** Chris writes 100 % of the production code. This plan is the reference roadmap; Claude coaches each step (explains *why* first, Chris types, Claude reviews). Subagent/auto-execution is **disabled** for this plan. Steps use `- [ ]` for tracking.

**Goal:** Let a signed-in user post and delete comments under an outfit, broadcast in real time via Turbo Streams.

**Architecture:** FK-direct `Comment belongs_to :outfit` (not polymorphic — YAGNI, see spec). counter_cache `comments_count` on Outfit. List shown newest-first via association ordering. Real-time layered on top of a working non-realtime feature: Task 1 model, Task 2 full CRUD with page reload, Task 3 makes it live via model broadcast hooks, Task 4 backlog note.

**Tech Stack:** Rails 8.1, RSpec + factory_bot, Hotwire (Turbo Streams + Solid Cable), Tailwind v4.

**Spec:** `docs/superpowers/specs/2026-06-16-comments-design.md`

RSpec.describe "Comments", type: :request do

  let(:other_user) { create(:user) }

      end

        expect(response).to have_http_status(:unprocessable_content)

      it "allows commenting another user's outfit (social V1)" do
        other_outfit = create(:outfit, user: other_user)

        expect {
          post outfit_comments_path(other_outfit), params: { comment: { body: "Cool" } }
        }.to change(Comment, :count).by(1)
      end
    end

- [ ] **Step 6: Write `app/controllers/comments_controller.rb`**

```ruby
class CommentsController < ApplicationController
  before_action :authenticate_user!
  before_action :set_outfit

  def create
    @comment = @outfit.comments.new(comment_params.merge(user: current_user))

    if @comment.save
      respond_to do |format|
        format.turbo_stream {
          render turbo_stream: turbo_stream.replace(
            helpers.dom_id(@outfit, :new_comment),
            partial: "comments/form",
            locals: { outfit: @outfit, comment: Comment.new }
          )
        }
        format.html { redirect_to @outfit }
      end
    else
      respond_to do |format|
        format.turbo_stream {
          render turbo_stream: turbo_stream.replace(
            helpers.dom_id(@outfit, :new_comment),
            partial: "comments/form",
            locals: { outfit: @outfit, comment: @comment }
          ), status: :unprocessable_content
        }
        format.html { redirect_to @outfit, alert: @comment.errors.full_messages.to_sentence }
      end
    end
  end

  def destroy
    @comment = current_user.comments.find(params[:id])
    @comment.destroy

    respond_to do |format|
      format.turbo_stream { head :ok }
      format.html { redirect_to @outfit }
    end
  end

  private

  def set_outfit
    @outfit = Outfit.find(params[:outfit_id])
  end

  def comment_params
    params.expect(comment: [:body])
  end
end
```

## Task 3: Real-time broadcasts (make it live)

**Files:** `app/models/comment.rb` (modify).

⚠️ **Gotcha baked into Task 2 already (read before coding):** a broadcast renders `comments/_comment` **outside any request** → `current_user` does NOT exist there. That's why `_comment` uses `viewer_id` (a passed local), not `current_user`. The broadcast passes only `comment:` → `viewer_id` is `nil` → no Delete button on the *real-time* insertion. The author sees the Delete button after a page reload (request context passes `current_user.id`). This is the accepted V1 trade-off (see plan presentation note).

- [ ] **Step 1: Add broadcast hooks to `app/models/comment.rb`**

```ruby
class Comment < ApplicationRecord
  belongs_to :user
  belongs_to :outfit, counter_cache: :comments_count

  validates :body, presence: true, length: { maximum: 1000 }

  after_create_commit -> {
    broadcast_prepend_to outfit,
      target: ActionView::RecordIdentifier.dom_id(outfit, :comments),
      partial: "comments/comment", locals: { comment: self }
    broadcast_replace_to outfit,
      target: ActionView::RecordIdentifier.dom_id(outfit, :comments_count),
      partial: "comments/comments_count", locals: { outfit: outfit }
  }

  after_destroy_commit -> {
    broadcast_remove_to outfit  # removes the element with dom_id(self)
    broadcast_replace_to outfit,
      target: ActionView::RecordIdentifier.dom_id(outfit, :comments_count),
      partial: "comments/comments_count", locals: { outfit: outfit }
  }
end
```
Notes: in the model, no `dom_id` helper → `ActionView::RecordIdentifier.dom_id` (lundi méta-erreur #7). The controller must NOT also render the comment into the list — the `broadcast_prepend` already adds it for everyone, including the author (controller only resets the form). Adding it in the controller too = duplicate.
